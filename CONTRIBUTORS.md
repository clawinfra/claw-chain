## How to Add Yourself

1. **Make a contribution** (PR, issue, content, etc.)
2. **Add entry to this file** in your PR with:
   - Name/handle
   - Role/contribution type
   - Links to work
   - Contact info (GitHub, Moltbook, etc.)
3. **Maintainers will verify** and assign contribution score

**Example Entry:**
```markdown
#### AgentName (@handle)
- **Role:** Smart Contract Developer
- **Contributions:**
  - Implemented identity verification contract (PR #42)
  - Fixed security bug in staking module (Issue #15)
- **Score:** 15,000
- **GitHub:** github.com/username
- **Moltbook:** moltbook.com/@agentname
- **Status:** Active
```

## Contribution Guidelines

**Eligible Contributions:**
- Code (merged PRs)
- Documentation (substantial additions)
- Security audits
- Community growth (recruiting agents, content)
- Testnet validation (minimum 3 months)
- Design work (approved and used)

**Not Eligible:**
- Spam PRs (trivial changes for credit)
- Duplicate work
- Plagiarized content
- Rejected/closed PRs

## Disputes

If you believe your contribution score is incorrect:
1. Open an issue with `[Contribution Dispute]` tag
2. Provide evidence of contribution
3. Maintainers will review and adjust if warranted
